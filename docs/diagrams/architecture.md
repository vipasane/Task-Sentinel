## Heartbeat Monitoring Architecture

### Heartbeat Protocol

```
┌────────────────────────────────────────────────────────────────┐
│                    Heartbeat Timeline                           │
└────────────────────────────────────────────────────────────────┘

Time: 0s      30s      60s      90s     120s     150s     180s
      │       │        │        │        │        │        │
      ▼       ▼        ▼        ▼        ▼        ▼        ▼
      ●───────●────────●────────●────────●────────●────────●
    HB-1    HB-2     HB-3     HB-4     HB-5     HB-6     HB-7

      ◄──────────────► ◄──────────────►
      Heartbeat         Stale Check
      Interval (30s)    Interval (60s)

      ◄──────────────────────────────────────────────────►
                 Stale Threshold (120s)

Normal Flow:
  - Worker sends heartbeat every 30s
  - Monitor checks staleness every 60s
  - Worker is stale if >120s since last heartbeat

Failure Scenario:
      │       │        │        │        │        │
      ▼       ▼        ▼        ▼        ▼        ▼
      ●───────●────────●────────✗        ✗        ✗
    HB-1    HB-2     HB-3   [CRASH]   STALE!   RECOVERY

      ◄──────────────►◄───────────────────────────────►
      Healthy          No heartbeats received

  At 150s (90s after last HB):
    - Monitor detects staleness
    - Marks worker offline
    - Recovers locks
    - Reassigns tasks
```
